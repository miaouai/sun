# 指南针旋转逻辑 Bug 修复报告

## 🐛 问题描述

**现象**: 当设备顺时针旋转时，指南针指针的旋转方向与实际地理方位相反。

**错误示例**:
- 手机朝东 (90°) → 红色指针指向东方 ❌
- 实际上北应该在设备的左侧（西边）

---

## 🔍 根本原因

### 参考系混淆

| 概念 | 含义 |
|------|------|
| **azimuth** | 设备顶部朝向的角度（以正北为 0°，顺时针） |
| **compass needle** | 应该始终指向地理北方的指针 |

**旧代码逻辑错误**:
```javascript
needle.style.transform = `rotate(${azimuth}deg)`;
```

这导致指针"跟随"设备一起转动，而不是"抵消"转动来保持指北。

---

## ✅ 修复方案

### 新算法

```javascript
// 核心公式：指针角度 = (360 - 设备角度) % 360
const pointerAngle = (360 - azimuth) % 360;
needle.style.transform = `translate(-50%, -50%) rotate(${pointerAngle}deg)`;
```

### 验证表

| 设备朝向 (azimuth) | 北在哪个方向？ | 指针应转至 | 计算验证 |
|-------------------|--------------|-----------|---------|
| 0° (北) | 前方 | 0° | (360-0)%360 = 0° ✅ |
| 90° (东) | 左侧 | 270° | (360-90)%360 = 270° ✅ |
| 180° (南) | 后方 | 180° | (360-180)%360 = 180° ✅ |
| 270° (西) | 右侧 | 90° | (360-270)%360 = 90° ✅ |

## 🎓 知识点

### 为什么需要反向旋转？

想象你站在一个圆盘上：
- 圆盘代表你的设备
- 你顺时针旋转了 90°（朝东）
- 为了让人物模型继续指北，它必须逆时针旋转 90°

这就是 `(360 - azimuth)` 的本质：**用相反的旋转抵消设备的转动**。
